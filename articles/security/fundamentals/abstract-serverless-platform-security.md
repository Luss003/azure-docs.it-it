# <a name="azure-functions-and-serverless-platform-security"></a>Funzioni di Azure e sicurezza della piattaforma senza server
## <a name="abstract"></a>Sunto
Per gestire i server la maggior parte delle aziende ha bisogno di molte risorse e tempo, cosa che aggiunge costi. Se le aziende possono usare meno risorse per gestire i server, riescono a concentrarsi sulla compilazione di applicazioni eccellenti.  

Con l'elaborazione senza server è possibile, perché l'infrastruttura necessaria per eseguire e ridimensionare le app è gestita automaticamente. L'elaborazione senza server è l'astrazione di server, infrastruttura e sistemi operativi. L'elaborazione senza server è basata sulla reazione a eventi e trigger che si verificano in tempo quasi reale nel cloud. 

In quanto servizio completamente gestito la gestione dei server e la pianificazione della capacità sono invisibili agli sviluppatori. Il framework senza server consente di sviluppare e distribuire applicazioni senza server mediante Funzioni di Azure. È un'interfaccia della riga di comando (CLI) che offre struttura e automazione per la compilazione di architetture sofisticate, senza server e basate su eventi, composte da funzioni ed eventi. Una funzione di Azure è un'unità indipendente dalla distribuzione, ad esempio un microservizio. È semplicemente codice, distribuito nel cloud, che viene spesso scritto per eseguire un singolo processo.

Nonostante i vantaggi, la sicurezza senza server ha propri fattori di rischio da affrontare. L'approccio senza server non introduce nuovi problemi di sicurezza, ma richiede un approccio ai problemi di sicurezza esistenti. Questo white paper è incentrato su questi argomenti relativi alla sicurezza: 
* Vantaggi di una piattaforma senza server
* Problemi di sicurezza nell'elaborazione senza server
* Problemi di sicurezza critici e mitigazioni nel contesto di Azure
* Protezione della piattaforma senza server Microsoft
